# Skills

Each skill is a folder under `app/skills/<skill_id>/`:

```
app/skills/<skill_id>/
  SKILL.md        required — frontmatter (name, description) + instructions body
  tools.py         optional — exposes TOOLS / SUB_AGENTS lists of ADK tools/sub-agents
  scripts/         optional — python/bash helper scripts used by tools.py
  resources/       optional — static data: json schemas, reference data, etc.
  artifacts/       optional — docs generated by or reference for the skill
```

`SKILL.md` format:

```markdown
---
name: skill-id
description: One line the root agent sees up front, to decide when this skill is relevant.
---

Detailed instructions for the agent to follow once this skill is triggered.
```

## How this loads

`app/skills/loader.py` scans every `SKILL.md` and reads only the frontmatter
(`name`/`description`) at startup — that's what gets baked into the root
agent's system prompt (see `load_skills()` in `app/skills/__init__.py`), so it
always knows what skills exist without paying for their full instructions in
every request's context.

The instructions body is only read when the agent calls the
`get_skill_instructions(skill_id)` tool — progressive disclosure. This is the
"trigger": the agent decides, from the name/description, that a skill is
relevant, then loads its full instructions on demand.

If a skill needs actual ADK tools or sub-agents (not just instructions), add
a `tools.py` to its folder with `TOOLS`/`SUB_AGENTS` lists — `load_skills()`
picks these up automatically via `register_skill()`. No changes to
`app/core/agent.py` are needed to add a new skill.
